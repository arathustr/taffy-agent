# UI 与动态设计

## 总体体验

Taffy Agent 的第一屏不是传统软件窗口，而是桌面上的塔菲角色。用户应该能一眼看出她现在是闲着、在听、在想、在执行，还是需要用户帮忙。

设计关键词：

- 活的：表情和动作随状态改变。
- 不挡路：默认小体积，任务需要时展开。
- 可读：状态、风险、结果都能看清。
- 可控：任何长任务都能暂停、取消、接管。

## 窗口形态

### Pet Mode

默认桌宠形态。

- 透明背景。
- 无边框、置顶、可拖动。
- 角色旁显示一条短气泡。
- 点击角色打开小型命令面板。
- 右键打开菜单：聊天、任务、设置、隐藏、退出。

### Focus Mode

执行复杂任务时自动或手动展开。

- 左侧或右侧停靠一条轻量任务面板。
- 角色仍然保留在面板边缘。
- 展示当前计划、工具调用、等待确认、输出摘要。

### Terminal Mode

面向 Codex、Shell、项目操作。

- 展示任务时间线、命令输出摘要、当前工作目录、最近变更。
- 不直接铺满终端噪音，只显示关键状态。
- 支持展开查看完整日志。

### Browser Assist Mode

浏览器自动化时使用。

- 展示当前网址、动作、等待条件和登录状态。
- 有“我来操作”“继续自动”“停止任务”三个明确入口。
- 截图缩略图只在需要确认时出现。

## 动态状态表

| 状态 | 角色表现 | UI 表现 | 触发条件 |
| --- | --- | --- | --- |
| idle | 呼吸、眨眼、小幅待机 | 无任务条或弱状态点 | 无任务 |
| listening | 轻微前倾、看向用户 | 输入波形/光标 | 语音或文本输入 |
| thinking | 眨眼变慢、轻晃 | “思考中”状态点、无长文解释 | LLM 推理 |
| planning | 拿出小面板/认真表情 | 计划列表生成 | 多步任务 |
| executing | 专注动作、键盘/工具感 | 工具调用流 | Shell/Codex/Browser 执行 |
| waiting_user | 注视用户、提示动作 | 确认卡片高亮 | 登录、验证码、风险操作 |
| success | 轻快动作、短庆祝 | 完成摘要 | 任务成功 |
| blocked | 困惑/皱眉 | 阻塞原因+可选操作 | 缺配置、登录失败、命令失败 |
| error | 轻微慌张但克制 | 错误摘要+重试/查看日志 | 异常 |

## 视觉层级

1. 角色本体：最高识别度。
2. 当前状态：一句话或一个图标即可。
3. 任务卡：只在有任务时展开。
4. 详细日志：默认折叠。
5. 设置面板：独立窗口，不挤在桌宠旁边。

## 角色资产路线

### 阶段 A：状态化占位

- 不使用 meme GIF 作为默认主体。
- 先用状态化 CSS/精灵占位跑通窗口、气泡、输入和状态变化。
- 每个状态用动作、表情、位移和气泡变化增强。

### 阶段 B：Live2D

- 标准 Cubism 模型优先：`model3.json`、`moc3`、texture、motion、expression。
- 状态映射到 expression/motion。
- 支持口型同步：TTS 音量驱动 mouth open。

### 阶段 C：高级演出

- 情绪驱动动作队列。
- 工具调用时显示小道具：浏览器、终端、代码、放大镜。
- 长任务期间有低频待机动作，避免完全静止。

## 气泡与任务卡

### 气泡内容

- 默认只显示当前最重要一句。
- 不连续刷屏。
- 长内容进入详情面板。

### 任务卡字段

- 任务名。
- 当前阶段。
- 正在使用的工具。
- 风险等级。
- 最近一步结果。
- 操作按钮：暂停、取消、详情、接管。

## 动画原则

- 动画服务于状态，不做纯装饰堆叠。
- 状态切换要快，长任务动作要慢。
- 错误状态不要吓人，给出可恢复感。
- 执行工具时角色动作和任务流必须同步，避免“角色在庆祝但任务失败”。

## UI 验收标准

- 用户 1 秒内能看出当前状态。
- 用户 3 秒内能找到暂停/取消。
- 长任务不会遮挡主要工作区。
- 无 LLM/API Key 时仍能打开 UI 并进入 Mock 模式。
- 窗口重启后恢复位置、缩放、置顶设置。
