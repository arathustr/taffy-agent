# 安全与权限设计

## 基本原则

Taffy Agent 是一个能操作电脑的 Agent，因此安全不是附加功能，而是核心体验的一部分。

原则：

- 默认最小权限。
- 用户可看见当前动作。
- 高风险动作需要确认。
- 密钥只保存在本地配置，不进日志，不进 prompt。
- 任何工具失败都要可恢复。

## 权限分级

| 等级 | 动作 | 默认策略 |
| --- | --- | --- |
| L0 | 闲聊、解释、状态显示 | 直接执行 |
| L1 | 读取公开网页、读取非敏感项目文件 | 允许，可在设置中关闭 |
| L2 | 写入项目文件、创建目录、下载文件 | 首次确认或按项目授权 |
| L3 | 运行命令、安装依赖、启动进程 | 需要确认，可建立信任目录 |
| L4 | 登录网站、提交表单、上传文件 | 每次确认或站点级授权 |
| L5 | 删除文件、付款、发布、推送、修改账号设置 | 每次强确认 |

## 强确认动作

这些动作不能静默执行：

- 删除、覆盖、移动大量文件。
- Git commit、push、force push、发布包。
- 支付、购买、订阅、退款。
- 修改账号密码、邮箱、权限、隐私设置。
- 发送邮件、私信、评论、公开发布内容。
- 执行来自网页的不可信命令。

## 密钥管理

### Taffy 自身密钥

`.env`：

```env
DEEPSEEK_API_KEY=
DEEPSEEK_BASE_URL=https://api.deepseek.com
```

要求：

- `.env` 加入 `.gitignore`。
- UI 设置页只显示是否已配置，不显示完整密钥。
- 日志中只显示密钥后四位或完全隐藏。

### Codex 密钥

- Taffy 不接管。
- Taffy 不要求用户输入 Codex API Key。
- Taffy 不读取 Codex 登录 token。
- Codex 配置由 Codex 自己管理。

## Policy Gate 输入

```json
{
  "taskId": "string",
  "tool": "browser|codex|shell|file|app",
  "action": "string",
  "risk": "low|medium|high|critical",
  "summary": "string",
  "sideEffects": ["string"],
  "requiresApproval": true
}
```

## 确认卡片

确认卡片必须显示：

- 要做什么。
- 会影响哪里。
- 是否可撤销。
- 为什么需要这一步。
- 允许、拒绝、手动处理三个选择。

## 日志安全

默认不记录：

- 密钥。
- 密码。
- cookie。
- 2FA。
- 完整网页隐私内容。

可以记录：

- 工具名。
- 动作摘要。
- 文件路径。
- 命令名和参数，必要时脱敏。
- 错误类型。
- token 用量。

## 隐私设置

用户应能控制：

- 是否启用语音输入。
- 是否保存会话记忆。
- 是否保存浏览器 session。
- 是否允许读取当前屏幕。
- 是否允许自动运行 shell。
- 是否允许 Taffy 调用 Codex。

## 安全验收标准

- 无 `.env` 时程序能启动并提示配置。
- 日志不含明文 API Key。
- L4/L5 动作必须出现确认卡。
- 用户能随时暂停/取消任务。
- 浏览器登录和 2FA 由用户接管。
