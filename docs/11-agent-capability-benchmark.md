# Agent 能力矩阵

## 目标

“不输 Hermes/OpenClaw”不能只理解为模型更强，而要拆成可实现、可测试、可恢复的系统能力。Taffy Agent 的目标是达到专业 Agent 工作流的能力密度，同时保留桌宠形态的低打扰体验。

## 能力分层

| 层级 | 名称 | 标准 |
| --- | --- | --- |
| L0 | Chat Bot | 能聊天、回答问题 |
| L1 | Tool User | 能调用单个工具 |
| L2 | Task Agent | 能拆解任务并连续调用工具 |
| L3 | Workbench Agent | 有记忆、权限、日志、复盘和失败恢复 |
| L4 | Operating Terminal | 能协同 Codex/浏览器/Shell 完成真实电脑工作 |
| L5 | Autonomous Workspace Partner | 能长期跟踪项目、主动发现阻塞、形成可复盘工作流 |

Taffy Agent 完整版目标：L4 稳定可用，部分 L5 能力可用。

## 核心能力矩阵

| 能力 | 完整版要求 | 验收方式 |
| --- | --- | --- |
| 意图路由 | 用户一句话识别聊天/浏览器/Codex/文件/Shell/设置/风险 | 50 条测试指令分类正确 |
| 结构化计划 | 多步任务生成 JSON 计划，不靠散文执行 | schema 校验通过 |
| 工具执行 | Browser/Codex/Shell/File 可独立失败、独立恢复 | 断网/缺工具/命令失败测试 |
| 复盘判断 | 每步工具结果由 Reviewer 判断继续/重试/询问/完成 | 人工构造失败输出 |
| 权限门 | 高风险动作强确认 | L4/L5 测试无绕过 |
| 任务状态 | 长任务可暂停、取消、恢复、失败总结 | 任务状态机测试 |
| 记忆 | 用户偏好、项目习惯、任务历史可保存可删除 | 设置页和数据库检查 |
| 可观测性 | 每次 LLM/工具/状态变化有 trace | 任务结束可导出记录 |
| Codex 适配 | 能生成高质量 handoff 并观察 diff/test/log | 真实项目修复场景 |
| 浏览器接管 | 登录/验证码/2FA 可暂停给用户处理并继续 | 登录场景演练 |
| UI 同步 | 角色状态与任务状态一致 | 手工和自动截图检查 |
| 模型路由 | Flash/Pro/Mock 可切换 | 无 Key、低成本、高质量三种模式 |

## 专业功能清单

### Planner

- 能把目标拆成步骤。
- 能标注每步工具和风险。
- 能生成停止条件。
- 能在失败后修订计划。

### Executor

- 工具调用有超时。
- 工具调用可取消。
- 工具结果结构化。
- 命令输出和 UI 输出分离。

### Reviewer

- 判断结果是否满足目标。
- 识别缺少信息。
- 决定是否重试或询问用户。
- 形成完成摘要。

### Memory

- 不把长期记忆塞满 prompt。
- 先检索相关记忆，再注入上下文。
- 用户能删除和禁用。
- 项目记忆和个人记忆分开。

### Observability

- 每个任务一个 trace 目录。
- 每个工具调用一个 call id。
- LLM 原始输出和清洗结果分开。
- 错误原因分类。

### Human-in-the-loop

- 风险确认。
- 登录接管。
- 页面截图确认。
- Codex 结果复核。
- 命令执行前确认。

## L5 后续能力

这些不要求首版完成，但架构要留位置：

- 周期性检查项目待办项/失败测试。
- 自动生成任务回顾。
- 多项目上下文切换。
- 根据用户偏好主动建议下一步。
- 对 Codex 输出做二次 review。
- 小型评测集，跟踪 Agent 能力回归。

## 不做的伪强大

- 不把 LLM 文本直接 eval 成命令。
- 不绕过登录验证。
- 不让角色台词掩盖失败。
- 不用无法复盘的长链黑箱自动化。
- 不把用户密钥或网页隐私发进无关上下文。
