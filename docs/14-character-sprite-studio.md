# Character Sprite Studio

## 定位

Character Sprite Studio 是 Taffy Agent 内置的角色像素精灵生成工具。目标是让用户放入任意人物设定主图后，自动生成可直接用于桌宠的透明像素风动画资源包，并尽量通过自动质检、自动重抽和自动对齐减少手动修帧。

它不是普通的“把图转像素风”工具，而是面向桌宠运行时的资产生产线：

- 输入：人物主图、角色描述、风格强度、目标尺寸、动作包选择。
- 输出：透明 PNG 帧、sprite sheet、预览动图、动画 manifest、质检报告。
- 使用：一键应用到桌宠状态机，或导出为可分享的角色资源包。

## 发布级目标

发布级版本必须满足：

- 可在无美术基础的情况下生成一套能跑的桌宠动画。
- 角色主特征在不同动作之间保持稳定，包括发型、发色、服饰色块、主要配饰、轮廓比例。
- 每个动作至少有自动对齐后的透明帧和 sprite sheet。
- 生成失败不是丢给用户修，而是进入自动诊断和重抽。
- 用户只需要在少数场景做选择：选图、选尺寸、选动作包、接受/重抽。
- 所有生成结果可追溯：记录输入图 hash、prompt、生成参数、质检分数、重抽次数。
- 公开发布时不内置未授权角色素材，只内置工具、空模板和示例占位资产。

## 用户流程

### 1. 导入角色

用户拖入一张或多张图片：

- 单张主图：最低输入，工具自动推断角色特征。
- 主图 + 表情图：更稳定，适合生成表情动画。
- 主图 + 侧面/背面：最佳输入，适合生成走路和转身。

工具先创建 `character-profile.json`：

- 角色名称。
- 主要色板。
- 发型、衣服、配饰、体型轮廓。
- 需要保留的视觉特征。
- 禁止漂移的特征。
- 目标像素尺寸和动作包。

### 2. 生成角色标准稿

先生成一张 canonical sheet，而不是直接生成动作帧。canonical sheet 用于锁定角色：

- idle front
- idle side
- idle back
- face neutral
- face happy
- face focused
- face upset
- color palette strip

这一步通过自动检测确认角色可用：

- 透明背景或纯色背景可分离。
- 角色居中完整。
- 没有额外人物或复杂道具污染。
- 像素轮廓清晰。
- 主色数量在目标范围内。

### 3. 生成动作

每个动作按“动作计划 -> 粗帧 -> 自动质检 -> 重抽坏帧 -> 对齐 -> 打包”的流程执行。

默认动作包：

| 动作 | 帧数 | 用途 |
| --- | ---: | --- |
| idle | 6 | 待机呼吸、轻微晃动 |
| listening | 6 | 等待用户输入、转头/眨眼 |
| typing | 8 | 调用 Codex 或处理任务时敲击/忙碌 |
| thinking | 8 | 思考、计划、检索 |
| executing | 8 | 执行工具、浏览器操作、跑命令 |
| waiting_user | 6 | 等待确认、登录、验证码接管 |
| success | 6 | 任务完成 |
| error | 6 | 失败、疑惑、需要用户处理 |
| sleep | 8 | 空闲较久 |
| dragged | 4 | 被拖动窗口时 |

### 4. 自动预览和选择

工具展示每个动作的循环预览：

- 正常播放。
- 2x 慢放。
- 透明背景和棋盘格背景切换。
- 锚点显示：脚底、中心、头部。
- 质检分数和问题标注。

用户可以：

- 接受整套动作。
- 只重抽某个动作。
- 锁定好帧，只重抽坏帧。
- 切换风格预设后重新生成。

### 5. 一键应用

资源包通过 `pet-sprite-manifest.json` 接入桌宠：

- 动作状态映射到 Agent 状态。
- 每个动作有 FPS、loop、anchor、hitbox。
- 桌宠窗口按透明 PNG 实际边界裁剪视觉区域。
- 工作台打开时仍可保持桌宠动画不中断。

## UI 设计

### 主入口

入口放在工作台的资产页，不放在默认桌宠小形态里。小形态只提供“换装/资产”快捷入口。

### 页面结构

- 左侧：输入和角色 profile。
- 中间：生成预览，含当前动作、帧序列、透明检查。
- 右侧：质量报告、重抽策略、导出选项。
- 底部：任务日志和每轮生成进度。

### 状态反馈

桌宠自身也要参与反馈：

- 正在识别主图：观察图片。
- 正在生图：拿画笔/敲键盘。
- 正在抠图：剪影闪烁。
- 正在对齐：脚底锚点提示。
- 生成成功：播放新动作预览。
- 生成失败：提示失败原因和下一轮自动处理。

## 最少手动修原则

手动修只作为兜底，不作为主流程。发布级默认启用以下自动化：

- 坏帧自动重抽：角色缺肢、头发漂移、透明残留、画布越界、帧间抖动超过阈值。
- 动作自动补帧：动作突变时插入中间帧或重排帧序。
- 自动脚底锚点：用角色轮廓下沿和连通域中心定位。
- 自动色板归一：抽取主色后统一量化，减少帧间颜色闪烁。
- 自动描边修复：透明边缘膨胀/腐蚀后重新像素描边。
- 自动尺寸归一：保持目标画布尺寸，禁止播放时动态撑大窗口。
- 自动生成报告：明确哪些帧被重抽、为什么重抽、最终是否达标。

允许用户手动调整的只有：

- 角色名和简短描述。
- 需要强调/禁止的外观特征。
- 动作包选择。
- 目标尺寸。
- 个别帧接受或重抽。

## 质量门槛

资源包标记为 `release-ready` 需要满足：

- 所有动作都有 `sheet.png` 和独立帧。
- 每帧透明背景 alpha 合格，边缘无明显底色残留。
- 每个动作锚点漂移小于配置阈值。
- 主色板漂移小于配置阈值。
- 角色主体没有被裁切。
- 每个动作至少一个完整循环无明显跳帧。
- manifest schema 校验通过。
- 预览图生成成功。

## 不做的事

- 不自动生成或模仿真人、未授权商业角色的发布素材。
- 不承诺单张图一次生成完美动画。
- 不将用户输入主图上传到未配置的第三方服务。
- 不把未授权素材打包进公开发行版。
- 不绕过生成服务的使用限制或版权策略。

## 首版实现范围

首版可以按两层交付：

### MVP

- 导入主图。
- 生成 canonical sheet。
- 生成 `idle`、`thinking`、`typing`、`success`、`error`。
- 自动抠图、对齐、打包。
- 导出 manifest。

### 发布级

- 完整动作包。
- 自动重抽坏帧。
- 质量报告。
- 资源包安装/卸载。
- 多角色库。
- 生成历史和版本回滚。
- 与桌宠状态机一键绑定。
